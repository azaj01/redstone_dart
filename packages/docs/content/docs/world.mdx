The World API lets you manipulate the Minecraft world—place blocks, spawn entities, play sounds, and more.

## How World Works

Unlike `Player` which wraps an entity ID, `World` wraps a **dimension identifier**. Minecraft has three dimensions by default:

- `minecraft:overworld`
- `minecraft:the_nether`
- `minecraft:the_end`

When you call methods on a World object, they make JNI calls to Java with the dimension ID and coordinates.

## Getting a World Reference

In block callbacks, you receive a `worldId` that you can use:

```dart
@override
ActionResult onUse(int worldId, int x, int y, int z, int playerId, int hand) {
  // Use World.overworld, World.nether, or World.end
  // Or work with the worldId directly in some APIs
}
```

For most operations, use the static dimension references:

```dart
World.overworld.setBlock(BlockPos(x, y, z), Block.diamondBlock);
```

## Placing and Reading Blocks

```dart
// Place a block
World.overworld.setBlock(BlockPos(100, 64, 100), Block.diamondBlock);

// Remove a block (place air)
World.overworld.setBlock(BlockPos(100, 64, 100), Block.air);

// Read what block is at a position
final block = World.overworld.getBlock(BlockPos(100, 64, 100));
// Returns a Block object
```

Block IDs follow Minecraft's `namespace:path` format. Vanilla blocks use `minecraft:` prefix.

## Spawning Entities

```dart
// Spawn a mob
World.overworld.spawnEntity('minecraft:pig', x, y, z);
World.overworld.spawnEntity('minecraft:zombie', x, y, z);

// Spawn lightning
World.overworld.spawnLightning(x, y, z);
```

Entity IDs follow the same `namespace:path` format as blocks.

## Particles and Sounds

Particles and sounds add feedback without affecting gameplay:

```dart
// Spawn particles
World.overworld.spawnParticles(Particles.flame, Vec3(x, y, z));
World.overworld.spawnParticles(Particles.heart, Vec3(x, y, z));
World.overworld.spawnParticles(Particles.explosion, Vec3(x, y, z));

// Play sounds
World.overworld.playSound(Vec3(x, y, z), 'minecraft:entity.experience_orb.pickup');
World.overworld.playSound(Vec3(x, y, z), 'minecraft:block.note_block.pling');
```

Sound IDs come from Minecraft's sound registry. You can find them in the Minecraft wiki or by looking at vanilla sound files.

## Explosions

```dart
// Basic explosion (no block damage)
World.overworld.createExplosion(Vec3(x, y, z), 4.0);

// TNT-like explosion (breaks blocks)
World.overworld.createExplosion(Vec3(x, y, z), 4.0, breakBlocks: true);

// Fiery explosion
World.overworld.createExplosion(Vec3(x, y, z), 4.0, causeFire: true);
```

The second parameter is power. TNT is 4.0, creepers are 3.0, the wither is 8.0.

## Time and Weather

```dart
// Set time (0-24000)
// 0 = dawn, 6000 = noon, 12000 = dusk, 18000 = midnight
World.overworld.timeOfDay = 6000;

// Weather
World.overworld.weather = Weather.rain;
World.overworld.weather = Weather.thunder;
World.overworld.weather = Weather.clear;
```

## Coordinate System

Minecraft uses a right-handed coordinate system:
- **X** — East (+) / West (-)
- **Y** — Up (+) / Down (-)
- **Z** — South (+) / North (-)

Y=64 is roughly sea level. The world extends from Y=-64 to Y=320 in modern Minecraft.

## Example: Teleporter Pad

A block that builds a platform and teleports the player:

```dart
@override
ActionResult onUse(int worldId, int x, int y, int z, int playerId, int hand) {
  final player = Players.getPlayer(playerId);
  if (player == null) return ActionResult.fail;

  final targetY = y + 50;

  // Build a glass platform
  for (var dx = -1; dx <= 1; dx++) {
    for (var dz = -1; dz <= 1; dz++) {
      World.overworld.setBlock(BlockPos(x + dx, targetY, z + dz), Block.glass);
    }
  }

  // Teleport player
  player.teleport(BlockPos(x, targetY + 1, z));

  // Effects
  World.overworld.spawnParticles(Particles.portal, Vec3(x.toDouble(), y + 1.0, z.toDouble()));
  World.overworld.playSound(Vec3(x.toDouble(), y.toDouble(), z.toDouble()), 'minecraft:entity.enderman.teleport');

  return ActionResult.success;
}
```
